## 📋 프로젝트 개요

### 🎯 핵심 아이디어

**전통적인 다마고치 게임을 AI 챗봇의 반응형 특성과 결합**

- AI가 펫의 개성과 감정을 자연스럽게 표현
- 사용자와의 자연어 대화를 통한 몰입감 있는 펫 케어
- MCP 서버로 게임 로직과 상태 관리의 안정성 확보

### 🏗️ 기술 아키텍처

```
[사용자] ↔ [Claude AI 챗봇] ↔ [MCP 서버] ↔ [게임 데이터베이스]

```

- **Claude AI**: 펫의 감정 표현, 자연어 상호작용, 스토리텔링
- **MCP 서버**: 게임 로직, 상태 관리, 데이터 정합성
- **데이터베이스**: 펫 정보, 사용자 상호작용 기록

## 🚀 AI 챗봇 활용의 차별화 포인트

### 💡 기존 다마고치 vs AI 다마고치

| 기존 다마고치 | AI 다마고치 |
| --- | --- |
| 정형화된 반응 | 상황별 맞춤 감정 표현 |
| 단조로운 케어 | 자연어 대화형 상호작용 |
| 제한된 이벤트 | 무한한 스토리 생성 |
| 획일적 성장 | 개성 있는 성장 패턴 |

### 🎭 AI 특화 기능 예시

**감정 표현**

> "코코가 조금 심심해 보여요. 큰 눈으로 당신을 쳐다보며 꼬리를 살살 흔들고 있어요."

**개성 반영**

## 🔧 MCP 서버 기능 명세

### 1️⃣ 펫 생명체 관리

- `create_pet()`: 펫 생성 및 초기 설정
- `get_pet_status()`: 실시간 상태 조회
- `update_pet_status()`: 시간 경과에 따른 자동 상태 변화

### 2️⃣ 기본 케어 시스템

- `feed_pet()`: 먹이 주기 (음식 종류별 효과)
- `play_with_pet()`: 함께 놀기 (다양한 활동)
- `clean_pet()`: 씻기기 및 청소
- `give_medicine()`: 치료 및 건강 관리

### 3️⃣ AI 반응형 특화 기능 ⭐

- `get_pet_personality()`: 펫 개성 데이터 (활발함, 수줍음, 호기심 등)
- `talk_to_pet()`: 사용자 메시지에 대한 펫 반응
- `get_pet_thoughts()`: 펫의 현재 생각/감정
- `record_interaction()`: 사용자 패턴 학습

### 4️⃣ 환경/이벤트 시스템

- `trigger_random_event()`: 랜덤 이벤트 발생
- `get_weather()`: 날씨가 펫 기분에 미치는 영향
- `visit_other_pet()`: 다른 펫과의 소셜 기능

## ❓ Q&A

**Q: 기존 다마고치 앱과 뭐가 다른가요?**
A: AI가 펫의 감정과 개성을 실시간으로 표현하여 훨씬 생동감 있고 예측 불가능한 상호작용이 가능합니다.

**Q: 기술적 난이도는 어떤가요?**
A: MCP 서버 개발은 기본적인 웹 개발 수준이며, AI 부분은 Claude가 담당하므로 적절한 난이도입니다.

**Q: 4주 안에 완성 가능한가요?**
A: MVP 수준의 핵심 기능은 충분히 구현 가능하며, 시연용 프로토타입으로 적합합니다.
